Codigo del sitio Web

En esta seccion explicare partes importantes del codigo, ya que muchos pensaran, que no es muy dificil hacer un sitio web, pero el echo de hacerlo, implica tener el rol de Diseñador UI, , Desarrollador Frontend y Maquetador Web. Y no solo esto si no toda la logica que hay por detras, por ejemplo el inicio de sesion, o al crear la interfaz de compra, para crear ese tipo de cosas, se utiliza cierta logica que hay que tener. Y eso es lo que estare explicando en esta seccion.

Inicio del Proyecto

Inicialmente cree la pagina de Productos, solo estaba haciendo unas pruebas acerca de como crear tarjetas, de forma dinamica, ¿Que quiere decir dinamica?, significa que uno no pone la informacion, si no que trae la informacion y apartir de eso, se crean automaticamente las tarjetas. Haciendo que todo sea mucho mas practico. La idea es que esa informacion sea cargada, por ejemplo de una api, una base de datos, etc. Pero en este proyecto, fue el caso, en este proyecto se utilizaron arrays para la manipulacion de la informacion, el array tenia cada uno de los objetos adentro, y cada objeto tenia la informacion de una sola tarjeta.

Ya esta la informacion en un array, ahora falta crear las plantillas de las tarjetas, para que a partir de codigo javascript, se cree cada una de las tarjetas, el template tiene un id, al cual llamamos por medio de codigo , pero no queremos el template, si no su contendio por eso se le pone la propiedad (.content).

Ahora lo que sigue es iterar sobre el array card y por cada elemento que esta en ese array se va a crear cada tarjeta, de acuerdo al bloque de codigo correspondiente, pero agi entra un tema que no se ha visto y es la Programacion Orientada a Objetos, (POO), este es un paradigma de programacion, que parte de consepto de "Objetos", es decir cada objeto tiene sus atributos y metodos. Utilice este paradigma para crear cada unas de las tarjetas, con sus correspondientes atributos y metodos.

Inicialmenete se queria hacer el proceso sin usar POO, pero la cuestion es que se necesitaba crearle un id a cada tarjeta, por ende recurri a la Programacion Orientada a Objetos, como se ve en la imagen tiene un atributo id que esta inicializado en 0, ¿Que pasa con ese id?, cada vez que se crea un Objeto, siempre se llama al constructor de la clase, es decir, si se crean tres Objetos de tipo Tarjeta, a la final se van a tener 3 tarjetas con diferentes id, ya que como se ve en el codigo del constructor, la Clase tarjeta tiene un atributo estatico, y cada vez que se crea un Objeto, este id aumenta en uno que seria este codigo (this.id = Tarjeta.id; Tarjeta.id++;).

En este metodo, esta el template, que se define asi: $template = document.getElementById("template").content, que ya se habia explicado que se le ponia la propieda content, para acceder al contenido, ahora adentro del metod se llama al $template y para buscar adentro del template se usa el metodo (querySelector()), como se ve en la imagen y el selector o lo que se este buscando debe estar adentro de la plantilla, la cual ya se mostro anteriormente.

En la imagen se ve que se le asignan valores de variables a cada uno de los elementos de la plantilla, es decir, al crear el objeto, se le pasan como parametros, (Los parametros son los que estan entre parentesis en el constructor de la clase) los valores de cada uno de los datos del array card, para que por medio del metodo cree cada una de las tarjetas, algo que toca hacer cuando se estan creando elementos asi, es crear un clon y pasarselo al fragmento, que posteriormente el fragmento es el que se añade al DOM (Document Object Model).

Ya estan añadidas las tarjetas al DOM, pero no es solo decir ya estan añadidas y ya, si no que van cosas de mano, como por ejemplo los estilos, estos se añaden por medio de una etiqueta link , que esta en el head de la pagina.

Y asi se crearia las tarjetas...

Menu de Navegacion

Para crear el encabezado negro, lo primero que busco es el buen posicionamiento, por ejemplo ese centrado y distancia que tiene cada elemento se hace con una propieda en especifica llamada FlexBox, puse el titulo y despues puse una caja con cada uno de los link, haciendo que en el encabezado hayan dos elementos, lo que permite mejor posicionamiento

Al header se le aplica una propieda llamada display: flex;, eso hace todos los elementos se vayan hacia el lado derecho de la pagina, pero todavia no estan separados, para separarlos por partes iguales, se le aplica al header la propiedad justify-content: space-around;, esto permite que cada elemento tenga una distancia igual en cada lado.

Ya estan los dos elementos separados, el titulo, de la caja de los link, pero siguen los link, estos estan hacia el lado derecho de la pagina, pero no tienen estilo ni nada, solo estan pegados, bueno pues lo que se hace es basicamente lo mismo que en el header, se le pone la propiedad display: flex; y se le pone la propiedad gap: 30px; que es una separacion que uno mismo pone la medida.
